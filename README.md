## 🛠️ Monorepo Architecture
DevBrand has evolved into a production-grade **Distributed Monorepo** for maximum scalability and domain isolation.

### 📂 Directory Structure
- **`apps/`**: The frontend nerve centers.
  - `web/`: The main high-fidelity platform (Next.js/Remix).
- **`modules/`**: Decoupled domain business logic.
  - `core/`: Event mesh orchestration and workflow engine.
  - `billing/`: Stripe integrations and pricing logic.
  - `repos/`: Repository synchronization and PR analysis logic.
- **`packages/`**: Reusable infrastructure SDKs.
  - `ai-sdk/`: Intelligent inference router with automatic failover (Claude/GPT-4o).
  - `telemetry/`: Distributed tracing and observability primitives.
  - `repo-intelligence/`: Deep code analysis and PR dissection engine.
- **`infrastructure/`**: Core shared services.
  - `database/`: Drizzle ORM schemas and migrations.
  - `cache/`: Global Upstash Redis orchestration.

## ⚙️ Technical Foundations
- **AI Orchestration**: Custom Inference Router with automatic fallback and cost optimization.
- **Event Mesh**: Asynchronous, event-driven architecture with built-in retry logic and Dead Letter Queues (DLQ).
- **Full-Stack**: TanStack ecosystem (Start, Router, Query).
- **Database**: CockroachDB/PostgreSQL + Drizzle.
- **Observability**: OpenTelemetry-ready distributed tracing across every service.

## ⚙️ Installation & Setup

### 1. Requirements
- Node.js 20+
- PostgreSQL instance (CockroachDB recommended)
- Redis (Upstash)

### 2. Quick Start
```bash
# Clone the repository
git clone https://github.com/bansalbhunesh/devbrand.git

# Install dependencies (Monorepo auto-hoisting)
npm install

# Setup environment variables
cp .env.example .env

# Push schema to database
npm run db:push

# Launch the entire platform
npm run dev
```
